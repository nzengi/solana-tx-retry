The Problem

Why transactions get dropped

Solana's high throughput is a double-edged sword. When the network is busy, even well-formed transactions regularly disappear into the void.

🌊

Network congestion

Leaders receive more transactions than they can fit in a block. Yours gets silently dropped — no error, no receipt, just silence.

⏳

Blockhash expiry

Every transaction carries a blockhash that expires after ~150 slots (~60s). Slow confirmation means your tx becomes invalid before it lands.

🔄

Leader rotation

Solana rotates block producers every 4 slots. Sending to the wrong validator means an extra gossip hop — losing precious milliseconds.

🎲

Thundering herd

When congestion hits, everyone retries at once. Without jitter, retry storms amplify the problem instead of solving it.

Features

Everything you need

Built for production Solana applications. Zero config to get started, fully tunable for power users.

🔁

Adaptive retry loop

Configurable max retries with exponential backoff. The engine owns the retry cycle — no double-counting from the RPC layer.

⚡

Blockhash auto-refresh

Detects TransactionExpiredBlockheightExceeded and automatically fetches a fresh blockhash and re-signs. Refresh attempts don't count against your retry budget.

🗺️

Leader schedule tracking

Fetches the full epoch leader schedule (432k slots) and maintains a live TPU address map. Route transactions directly to the upcoming leader.

💾

Preflight cache

Simulation results are cached by transaction message bytes (blockhash-independent). Re-signed transactions get instant cache hits, skipping redundant RPC simulation calls.

🎲

Jitter to prevent herd

Each backoff delay has ±20% random jitter added. This spreads retry storms across time, reducing amplification during network congestion events.

🛡️

Terminal error detection

On-chain rejections (e.g. insufficient funds, bad signature) are immediately classified as terminal and not retried — no wasted attempts.

📊

Rich stats output

Every successful send returns attempt count, blockhash refresh count, leader routing flag, and total elapsed time. Instrument your application with zero extra code.

🔧

Three built-in presets

default — balanced. high_throughput — max speed, no preflight. safe — always simulate, long timeout. One line to switch.

🌐

TypeScript wrapper

Drop-in replacement for sendAndConfirmTransaction from @solana/web3.js. Same API surface, retry superpowers underneath.

Usage

Start in 5 lines

Works with your existing Solana code — no migration required.

Rust
TypeScript
Presets
use solana_tx_retry::{RetryEngine, RetryConfig};

// Create engine with defaults (10 retries, 400ms base delay)
let engine = RetryEngine::new(Arc::clone(&rpc), RetryConfig::default());
engine.initialize().await?;   // fetches leader schedule

// Build your transaction as usual
let tx = Transaction::new_signed_with_payer(&[ix], ...);

// Send — engine handles retries, blockhash refresh, backoff
let stats = engine.send_with_retry(tx, &payer).await?;

println!("✓ {} (attempts: {})", stats.signature, stats.attempts);
import { sendWithRetry } from 'solana-tx-retry';

// Drop-in for sendAndConfirmTransaction
const stats = await sendWithRetry(
  connection,
  transaction,
  [payer],
  { maxRetries: 10, commitment: 'confirmed' }
);

console.log(`✓ ${stats.signature} — ${stats.attempts} attempt(s)`);
use solana_tx_retry::RetryConfig;

// ── Balanced (default) ─────────────────────────────────
let config = RetryConfig::default();
// max_retries: 10, base_delay: 400ms, preflight: true

// ── Maximum throughput ─────────────────────────────────
let config = RetryConfig::high_throughput();
// max_retries: 20, base_delay: 200ms, preflight: false

// ── Safety first ───────────────────────────────────────
let config = RetryConfig::safe();
// max_retries: 5, preflight: always, timeout: 60s

// ── Custom ─────────────────────────────────────────────
let config = RetryConfig {
  max_retries: 15,
  base_delay: Duration::from_millis(300),
  ..RetryConfig::default()
};
Install

Get started now

Available as a Rust crate (via Cargo) and a TypeScript package (via npm).

Rust (Cargo.toml)

solana-tx-retry = "0.1"

TypeScript / Node.js

npm install solana-tx-retry @solana/web3.js

Run devnet example

cargo run --example basic_transfer
